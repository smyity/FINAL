# Итоговый проект курса «DevOps-инженер с нуля»

## Описание проекта и выбор темы
**Тема итоговой работы** - Базовая DevOps‑инфраструктура в Yandex.Cloud

**Инструменты для выполнения итоговой работы**

- аккаунт в Yandex.Cloud, доступ к панели управления и к созданию ресурсов;
- установленный Terraform;
- установленный Git и учётная запись на GitHub/GitLab;
- SSH‑клиент для подключения к виртуальной машине;
- Docker на локальной машине (по желанию) для локальной проверки образа.
 
### Описание проекта
В итоговой работе вы должны продемонстрировать упрощённый DevOps‑цикл для тестового веб‑приложения:

1. В облаке Yandex.Cloud развёрнута инфраструктура: сеть (VPC), подсеть и виртуальная машина, доступная из интернета по IP‑адресу, и backend для Terraform на базе S3‑совместимого хранилища.
2. На виртуальной машине установлен Docker (и при необходимости Docker Compose), система готова к запуску контейнеров.
3. Есть тестовое приложение (например, nginx, отдающий статическую страницу), для него подготовлен Dockerfile и compose.yaml, а собранный Docker‑образ опубликован в выбранном реестре (Docker Hub или Yandex Container Registry).
4. Настроен CI/CD‑pipeline (GitHub Actions, GitLab CI или аналог), который:
   - при каждом коммите собирает Docker‑образ и отправляет его в реестр;
   - при коммите в основную ветку (например, main) разворачивает новую версию приложения на виртуальной машине (обновляет контейнер).
5. Подготовлена документация (README и отчёт), в которой вы описываете архитектуру решения, последовательность шагов, используемые инструменты и инструкции по повторному развёртыванию.

Результат должен быть воспроизводимым: другой человек, следуя вашей документации, должен суметь развернуть инфраструктуру и приложение с нуля.
 
### Инструкция по выполнению работы

1. **Подготовка облачной инфраструктуры**
   - Создайте новый репозиторий для Terraform‑конфигурации.
   - Скачайте и установите Terraform, если ранее этого не делали.
   - Настройте аутентификацию Terraform в Yandex.Cloud (сервисный аккаунт, авторизационные данные).
   - Создайте S3‑bucket в Yandex.Cloud и настройте backend для хранения Terraform‑состояния.
   - Опишите в Terraform создание VPC, подсети и виртуальной машины.
   - Проверьте конфигурацию командой terraform plan, затем примените её с помощью terraform apply.
   - Убедитесь, что ВМ создана и доступна по SSH.
 
2. **Установка Docker на виртуальной машине**
   - Подключитесь по SSH к созданной ВМ.
   - Установите Docker (и при необходимости Docker Compose):
     - либо вручную по официальной инструкции;
     - либо подготовьте Ansible‑playbook и примените его.
   - Проверьте, что Docker работает: выполните docker version и запустите тестовый контейнер.
 
3. **Подготовка тестового приложения**
   - Создайте отдельный репозиторий для тестового приложения.
   - Напишите простое приложение или статический сайт (можно на nginx), которое будет явно показывать, что деплой успешен (например, страница «DevOps диплом: ваше имя»).
   - Создайте Dockerfile для сборки образа приложения.
   - Создайте compose.yaml для запуска приложения через Docker Compose (если вам нужен больше чем один контейнер, например, приложение + БД).
   - Локально соберите образ и протестируйте его запуск.
 
4. **Публикация образа в реестре**
   - Зарегистрируйтесь в Docker Hub или создайте Yandex Container Registry.
   - Настройте логин к реестру.
   - Соберите образ и отправьте его в реестр (docker push или через CI).
 
5. **Настройка CI/CD**
   - В репозитории приложения создайте конфигурацию CI/CD (например, .github/workflows/ci‑cd.yml для GitHub Actions).
   - Добавьте шаги:
     - сборка Docker‑образа;
     - отправка образа в реестр;
     - деплой на ВМ (через SSH, Ansible или другой выбранный вами способ).
   - Настройте секреты (учётные данные для реестра и доступа к ВМ) в настройках репозитория.
   - Проверьте, что при коммите в ветку main pipeline успешно проходит и приложение обновляется на ВМ.
 
6. **Документация и финальная проверка**
   - Соберите все ссылки на репозитории и реестр в один README.
   - Опишите архитектуру, шаги развёртывания и проверки работоспособности.
   - Убедитесь, что при terraform destroy инфраструктура корректно удаляется, а при повторном terraform apply и настройке CI/CD всё снова поднимается и работает.
 
### Структура итоговой работы:
   - Подробный README с проектированием инфраструктуры и решениями, которые принимали при проектировании.
   - 3 репозитория на GitHub:
     - репозиторий с конфигурационными файлами Terraform;
     - репозиторий с конфигурацией Ansible;
     - репозиторий с Dockerfile, conpose.yaml тестового приложения и ссылка на собранный docker image.

### Порядок защиты итоговой работы
Защита итогового проекта проходит в письменном виде: вы сдаете работу через кабинет студента, эксперт проверяет вашу презентацию, оставляет комментарии.
На проверку дипломной работы эксперту дается 14 рабочих дней с момента загрузки в кабинет.

При необходимости дополнительной проработки, ваша работа будет отправлена на доработку, о чем вы получите уведомление через кабинет студента. Эксперт приложит детальные комментарии по тем пунктам, которые требуется дополнить.

У вас будет 7 рабочих дней на внесение правок. Не забудьте загрузить работу и отправить ее на повторную проверку.

### Критерии оценивания дипломной работы
Работа оценивается по критериям:

**Зачёт:**

1. Архитектура и инфраструктура.
   - Инфраструктура полностью описана в Terraform.
   - ВМ в Yandex.Cloud создаётся и удаляется командами Terraform и доступна по SSH.
2. Docker и окружение
   - Docker установлен, тестовый контейнер запускается.
   - (Если есть Ansible) Playbook отрабатывает без критических ошибок.
3. Контейнеризация приложения
   - Есть Dockerfile и compose.yaml.
   - Приложение в контейнере работает и доступно по HTTP.
   - Образ опубликован в реестре, ссылка указана.
4. CI/CD‑pipeline
   - При коммите образ собирается и отправляется в реестр.
   - При коммите в основную ветку выполняется деплой на ВМ.
   - Секреты не хранятся в открытом виде в коде.
5. Документация и оформление
   - README понятен, содержит описание архитектуры, последовательность действий логична.
   - Есть ссылки на репозитории и образ, по инструкции можно воспроизвести проект.
 
**На доработку:**
1. Архитектура и инфраструктура в Yandex.Cloud
   - Инфраструктура создаётся вручную.
   - Код неполный или не запускается.
   - ВМ невозможно использовать (нет доступа, неправильная сеть).
2. Установка Docker и подготовка окружения
   - Docker не установлен или не запускается.
   - Шаги установки не описаны и их нельзя повторить.
3. Контейнеризация приложения
   - Контейнер не запускается или приложение не отвечает.
   - Образ не опубликован или нет ссылки.
4. CI/CD‑pipeline
   - CI настроен частично (сборка есть, но нет публикации или деплоя).
   - Автоматического деплоя нет, всё выполняется вручную.
   - Секреты закоммичены в репозиторий.
5. Документация и оформление
   - README фрагментарный, непонятно, как развернуть решение.
   - Отсутствуют ссылки на часть репозиториев или они недоступны.
 
**Незачёт:**\
Работа выполнена не в полном объёме. Все попытки доработки израсходованы.

## Решение

Структура:
```
├── .env                   # переменные для docker compose
├── compose.yml
├── README.md
├── application
│   ├── back.py            # Принимает метрики и пишет в ClickHouse
│   ├── Dockerfile         # Собирает Python-приложение
│   ├── front.py           # Запрашивает интервалы у бэка и рисует графики
│   └── supervisord.conf   # Управляет процессами бэка и фронта
├── gitlab
│   ├── config
│   ├── data
│   └── logs
└── terraform(+ansible)
    ├── cloud-init.yml
    ├── global_variables.tf
    ├── deploy_infrastructure.yml # ansible playbook для установки docker и deploy инфраструктуры
    ├── main.tf
    └── terraform.tfvars          # вот здесь все переменные, которые нужно менять
```

### Step by step

1. Клонировать репозиторий проекта:

   ```
   git clone https://github.com/smyity/FINAL.git
   ```

2. Перейти в директорию terraform:

   ```
   cd terraform
   ```

3. В файле *terraform.tfvars* поменять значения переменных на свои.
4. Сделать инициализацию:

   ```
   terraform init
   ```
5. Запустить:

   ```
   terraform apply
   ```
